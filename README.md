# Hello 

# Это один из веб серверов на GO
# Идея его создания такая: мне нужно было создавать тестовые эндпоинты для базы данных для разработки одного из приложений. 
# Мне не хотелось каждый раз перекомпилировать проект при написании нового запроса, поэтому
# решено было сделать так, чтобы сервер был простой, мог обрабатывать любые запросы, из этого вышла такая структура

# 1. В папке sql есть папки /get, /noparams, /post , в каждой из них лежат файлы с запросами к базе.
# /noparams - запросы без параметров
# /get - отработка get запроса, параметры берутся из него
# /post - отработка post запроса, параметры берутся из его тела
# 
# 2. Договоренности 
# 2.1 Для get запроса параметры в sql прокидываются по принципу первый = $1, второй = $2 ... , все просто. Имена параметров при этом не важны, обзывайте как хотите
# 2.2 Для post запроса берется тело с параметрами и значениями, после чего происходит сортировка по имени параметров, а после сортировки происходит проброс в sql по принципу a = $1, b = $2, c = $3 ... , при этом особого внимания на имена не обращается, так что можете писать их просто в алфавитном порядке для удобства
#

# 3. Подключение к БД
# Используйте .env файл для настройки подключения к базе данных Postgres. 
#

# 4. Создание эндпоинтов. 
# Сервер компилируется один раз, запускается. 
# Для создания нового эндпоинта - создайте .sql файл в соответствующей папке. 
# Сервер автоматически подхватит его и станет отвечать на соответствующий запрос
#

# 5. Примеры
# Использование простое - мы обращаемся к приложению по адресу http://localhost:APP_PORT/noparams/my , по запросу будет выполнен запрос, записанный в sql/noparams/my.sql . 
# для get будет http://localhost:APP_PORT/get/myget?a=1&b=2 , выполнится запрос sql/get/myget.sql , пробросив параметры 
# для post то же самое, только в тело запроса вписать {"a":"1", "b":"2"}
# 

# 6. Польза в одном - если надо добавить поле выборки из БД, то меняй sql и используй дальше, ничего не перезапуская. 
#

# P.S. Для прода я использую другой сервер, где sql вшивается в сервер через embed , это будет быстрее работать, но при разработке требуется именно уклон в исследование. Так что сервер для разработчиков.
